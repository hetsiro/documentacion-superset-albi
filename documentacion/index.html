Superset + SQL Server

Guía completa para instalar Apache Superset con SQL Server en Docker, con actualización automática de datos cada hora y consultas instantáneas.

Apache Superset SQL Server 2019 Docker Redis Cache Windows
Arquitectura

Superset y SQL Server corren en la misma red Docker, eliminando la latencia de red y logrando consultas casi instantáneas.

SQL Server productivo (54.244.218.18)
    ↑ Linked Server (AMECO_DEV)
SQL Server Docker (sqlserver:1433)
    → Job cada 1 hora: TRUNCATE + INSERT
    → Tabla: AmecoHubIntegracion.dbo.AmecoDashboards
        ↑ misma red Docker (instantáneo)
Apache Superset (localhost:8088)
⚡
Con Superset y SQL Server en la misma red Docker, las consultas pasan de ~5 segundos a casi instantáneas.
Requisitos
  • Docker Desktop instalado y corriendo (ícono verde)
  • SQL Server Management Studio (SSMS)
  • Acceso al servidor SQL productivo
1 Crear carpeta y archivos de configuración

Crear la carpeta del proyecto

bash
mkdir C:\superset
cd C:\superset

superset_config.py

Dockerfile

dockerfile
FROM apache/superset:latest

USER root
RUN apt-get update && apt-get install -y python3-pip && \

      - ./superset_config.py:/app/pythonpath/superset_config.py

      - redis
      - db
      - sqlserver

volumes:
  db_data:
  sqlserver_data:
2 Construir y levantar los servicios
bash
cd C:\superset
docker compose up -d --build

Esperar unos minutos a que descargue todo y construya la imagen.

⚠️
Si reconstruís la imagen con --build, debés volver a ejecutar los comandos del Paso 4.
3 Crear la base de datos y el usuario admin

Ejecutar los 3 comandos uno por uno, esperando a que cada uno termine:

bash
docker compose exec superset superset db upgrade
bash
docker compose exec superset superset fab create-admin \
  --username admin \
  --firstname Admin \
  --lastname Admin \
  --email admin@admin.com \
  --password admin
bash
docker compose exec superset superset init
✅
Una vez completado, Superset estará disponible en http://localhost:8088 con usuario admin y contraseña admin.
4 Configurar SQL Server Docker

Conectarse desde SSMS a localhost,1434 con usuario sa y password AmecoSQL2024!

sql
-- Crear la base de datos
CREATE DATABASE AmecoHubIntegracion;

-- Habilitar consultas distribuidas
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure 'Ad Hoc Distributed Queries', 1;
RECONFIGURE;
5 Crear Linked Server al servidor productivo

Desde SSMS conectado a localhost,1434:

sql
EXEC sp_addlinkedserver
    @server = 'AMECO_DEV',
    @srvproduct = '',
    @provider = 'SQLNCLI',
    @datasrc = '54.244.218.18'

EXEC sp_addlinkedsrvlogin
    @rmtsrvname = 'AMECO_DEV',
    @useself = 'FALSE',
    @rmtuser = 'sa',
    @rmtpassword = 'TuPasswordAqui'

Verificar que funciona:

sql
SELECT * FROM OPENQUERY(AMECO_DEV, 'SELECT @@SERVERNAME')
ℹ️
Si devuelve el nombre del servidor productivo, el Linked Server está funcionando correctamente.

Agregar o eliminar columnas

Checklist — agregar columna nueva

  1. Identificar la tabla fuente en AMECO_DEV y verificar si tiene columna id separada del nombre — si no tiene id, agregar tanto el id como el nombre para que el catálogo funcione correctamente.
  2. Agregar el LEFT JOIN y las columnas al SELECT del Job en index.html (en los dos lugares: PASO 1 y PASO 2).
  3. Ejecutar el ALTER TABLE en SSMS para agregar la columna a la tabla existente (ver abajo).
  4. Actualizar el Job en SSMS con el nuevo SQL y ejecutarlo manualmente para poblar los datos.
  5. Ejecutar ALTER VIEW dbo.v_OrdenesTrabajo para refrescar los metadatos de la vista.
  6. Agregar la columna a cols_meta.js con campo, original, tabla, tipo y grupo.
  7. Agregar la entrada al array LOOKUPS en generar_lookup.py y correr generar_lookup.bat para regenerar el catálogo.
  8. En Superset → Dataset → Sync columns from source para que aparezca en los charts.

Agregar columna nueva

sql — SSMS localhost,1434
ALTER TABLE AmecoHubIntegracion.dbo.AmecoDashboards
ADD NuevaColumna VARCHAR(100)

Luego en Superset → Dataset → Sync columns from source

Eliminar columna o cambio estructural grande

sql
DROP TABLE IF EXISTS AmecoHubIntegracion.dbo.AmecoDashboards
🚨
Después de un DROP TABLE hay que ejecutar el job manualmente y reimportar el dataset en Superset, ya que se pierden las referencias.
